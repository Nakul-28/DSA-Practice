## Parameterized Recursion
- Definition: The recursive function carries extra parameters (often called an accumulator) to keep track of the intermediate state or the running result.
- How it Works: The function computes the partial result, passes it forward to the next call, and usually returns the final result directly at the base case.
- Key Characteristic: Often used in tail-recursive functions to save memory, as it computes results before the recursive call.
- Example **(Factorial)**: $fact(n, acc) \rightarrow fact(n-1, n*acc)$. The acc holds the intermediate factorial product. 

## Functional Recursion
- Definition: The function is defined in terms of itself to solve a problem by breaking it into smaller sub-problems.
- How it Works: The function calls itself, and when it returns, the result is used to calculate the final result of the current call.
- Key Characteristic: More descriptive/mathematical; it often builds the solution while unwinding the stack.
- Example **(Factorial)**: $fact(n) \rightarrow n * fact(n-1)$. The result is computed after the $fact(n-1)$ call returns.

## Sum of first n natural numbers
### Parameterized Recursion
```java
class Main{
    public static int sum(int n, int acc){
        if(n == 0){
            return acc;
        }
        return sum(n-1, acc + n);
    }
    public static void main(String[] args){
        int n = 5;
        System.out.println(sum(n, 0)); // Output: 15
    }
}
```
### Functional Recursion
```java
class Main{
    public static int sum(int n){
        if(n == 0){
            return 0;
        }
        return n + sum(n-1);
    }
    public static void main(String[] args){
        int n = 5;
        System.out.println(sum(n)); // Output: 15
    }
}
```
